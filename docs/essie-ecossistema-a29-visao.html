Ecossistema A-29 + Essie

Visão estratégica e plano de modularização

Autor: João André  |  Data: julho de 2026  |  Repos: varejosync (P38) → a29-erp + Essie

1. Resumo dos objetivos

  1. Agenda e rotina — Compromissos, contas vencendo e tarefas num só lugar (protótipo já no P38).
  2. Essie (S) — App separada: secretária inteligente (rotina, preparação, antecipação).
  3. Um núcleo, várias portas — Separar experiências sem separar a verdade dos dados.
  4. A-29 — Vercel + Supabase como instrumento sem “tubos com vazamento”.
  5. Rotinas prontas — Workflows, reuniões com pauta, agentes por contexto (compras, financeiro…).
  6. Acções reais — WhatsApp, e-mail, automação (n8n); Cursor constrói, Essie opera.

2. O problema

SintomaCausa
P38 funciona mas pesaUm app para vendas, estoque, financeiro, logística
Querer “partir” módulosCada função precisa de interface focada
Tubos com vazamentoApps separados sem base única
Cansaço diárioReinventar o dia sem rotinas
Reuniões fracasChegar sem pauta

Solução: Supabase (uma verdade) + interfaces modulares + Essie (rotina e acção).

4. Essie — visão de produto

Essie não é Monday nem ERP. É organização e execução da rotina.

Faz

  • Resumo do dia; rotinas por função
  • Preparação de reuniões com pauta (agente)
  • Assistentes por módulo (ex.: compras → confirmar embarque)
  • Acções com aprovação (WhatsApp, e-mail, registo no A-29)

Não faz

  • Substituir PDV, stock ou lançamentos oficiais
  • Duplicar dados do negócio

Integrações (fases)

  1. Supabase + e-mail + Google Calendar
  2. n8n + WhatsApp (Zenvia ou Twilio)
  3. Voz / chamadas (opcional)

Regras anti-vazamento

  1. Nenhum módulo guarda cópia oficial de produto, pedido ou lançamento.
  2. Escrita sempre no Supabase via APIs partilhadas (packages A-29).
  3. Perfis + RLS definem módulo e acções.
  4. Essie lê o mesmo banco; agenda em agenda_item.
  5. Eventos entre módulos via base (venda → lançamento → stock), não sync manual.

6. Fases de implementação

Fase 0 — Hoje (P38): Agenda no menu; migration Supabase; P38 estável.
Fase 1 — Fundação A-29: Supabase, auth, APIs partilhadas, deploy Vercel.
Fase 2 — Módulo C: Financeiro operacional (só lançamentos) — primeira porta.
Fase 3 — Módulo A: Frente de caixa / PDV dedicado.
Fase 4 — Módulo B: Supply + rotinas Essie (embarque, WhatsApp).
Fase 5 — Módulo D: Gerencial completo.
Fase 6 — Essie: App separada; rotinas v1; n8n + WhatsApp.
Fase 7 — Corte Base44: P38 manutenção; A-29 + Essie activos.

8. Critérios de sucesso

  • Caixa não vê menus gerenciais.
  • Lançamento em C aparece em D — mesmo ID.
  • Venda em A baixa stock em B sem export.
  • Essie e módulo B usam os mesmos pedidos de compra.
  • Ninguém “abre o sistema gigante para procurar”.

9. Posicionamento

A-29: Um negócio, uma base, a interface certa para cada função.

Essie: As rotinas do teu dia, ligadas ao teu negócio. Menos improviso, mais preparação.
